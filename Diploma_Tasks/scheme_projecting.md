# Задача №N - проектирование системы 
## Процесс решения

Создание сложных систем начинается с разработки требований и концептуальных схем. Такой подход упрощает дальнейшую разработку системы, так как любые изменения до воплощения задумки в код можно применить к схеме и спрогнозировать возможные негативные последствия, что может спасти проект от ненужных и лишних правок.
По первоначальной идее планируется предоставить пользователям выбирать документы для заполнения из главного меню, где будут расположены кнопки, при нажатии на которые будет производится переход на страницы заполнения. Для повышения удобства заполнения ддокументы будут разделены на отдельные страницы с полями для заполнения. Каждое окно пользовательской формы в Qt это отдельный настраиваемый класс со своими функциями. 
Переход со страницы на страницу подразумевает наличия функций по переходу в обе стороны, вперёд и назад. Также важно внести функцию по отмене заполнения документа, которая также будет переносить пользователя обратно в главное меню. Таким образом стандартный класс страницы имеет следующий вид:

```
class MainWindow {
  - ui: Ui::MainWindow*
  + MainWindow()
  + ~MainWindow()
  + on_clicked_push()
  + on_click_back()
  + on_click_cancel()
}
```
Таким образом можно выстроить цепочку из подобных классов с настройкой переходом между ними и к главному меню.
![Схема 2 системы](//Diploma_Tasks/Images/firstVersionOfDiagramm22.png)
Каждый новый класс наследуюет базовые функции для своего существования от QMainWindow. При создании новых классов, каждый имеет первоначально диструктор, отображение. Как заявлялось выше, каждый класс имеет определённый набор методов для навигации:
-  on_clicked_switchPage() - переносит но нажатию на следующую страницу
- on_click_back() - переносит на предыдущую страницу при нажатии
- on_click_cancel() - отменят заполнение документа и переносит в главное меню

Цель системы - предоставить возможность заполнения шаблонных документов. Следовательно, для заполнения необходимы соответствующие механизмы: сохранение данных для возможности редактирования и замены в шаблоне.
