# Задача №4 - выбрать технологический стек для разработки

## Процесс решения

Проект сервиса планируется делать с возможностью использования не только на личных компьютерах, а также мобильных устройствах. Данную цель можно достичь несколькими путями: создать универсальный сервис, где каждая его версия будет реализована для конкретного устройства, однако этот вариант не может быть выполнен из-за сроков сдачи MVP-версии проекта. Использование библиотеки Qt и TreeFrog фрйемворка для C++ можно избежать множества производственных проблем, например ручного создания backend части сервиса, так как она уже создана разработчиками и работает без проблем из коробки.

Для разработки программной части проекта был выбран язык программирования С++ и библиотека Qt.
Выбор данного языка программирования обоснован б**о**льшим контролем над процессами внутри исполняемого кода. Из преимуществ можно выделить следующие пункты:

- открытость языка и отсуствие принадлежности к компании распространителю, как например Java от Oracle
- фиксированное выделение памяти под рабочие задачи
- быстрота работы
- тип языка, а именно компилируемый
- поддержка принципов ООП

Для создания и управления базой данных и таблицами внутри выбрали SQLite3, так как с него можно без особых трудностей перейти на другие СУБД, например PostgreSQL. На данный момент потребности сервиса в более сложной базу данных отсуствует и внутренние возможности SQLIte3 покрывает все требования. Данная версия не требует к себе отдельного сервера для работы, что уменьшает потенциальную поверхность атаки на сервис со стороны нарушителей, отсуствием лишнего интерфейса для взаимодействия. Безопасность веб-приложений - сложный и комплексный процесс.
