# Задача - составление функциональных и нефункциональных требований

## Процесс решения

### Причины составления:
Первопричина для составления чёткого списка требований к системе - систематизация предстоящего процесса разработки. Без явного списка сформулированных требований нет ясных ориентиров для разработки. Список требований содержит критерии и рамки для разработчика, помогая достичь поставленной цели и не выходить за пределы поставленных задач в реализации функционала.

### Неформазилованные требования:
- Система должна предоставлять возможность заполнения бланков кинологических мероприятий для последующей печати и простановки подписи;
- Система должна сохранять прогресс заполнения документа при выходе из браузера и предлагать продолжить заполнения после повторного захода при условии сохранения открытой вкладки;
- Система должна выдавать файл на скачивание не позже 1 минуты после завершения заполнения;
- Система должна предоставить возможность прекращения заполнения документа при необходимости без нужды в сохранении файла с уже заполненными полями;
- Система должна предоставлять функционал по заполнению бланков только авторизованному пользователю;
- Система должна проверять пароль на соответсвие парольному стандарту и отклонять слишком простые варианты;
- Система не должна хранить персональные данные пользователей в открытом виде, либо вовсе не хранить таковых;
- Система должна сохранять информацию в полях после первой выдачи готового файла для возможности редактирования информации и составления нового файла с исправленными ошибками;
- Система должна предоставить возможность очистки текстовых полей по нажатию кнопки;
- Система должна предоставить возможность обработки голоса пользователя и перевода речи в текст при заполнении полей;
