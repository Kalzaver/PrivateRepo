# Задача №7 - выбрать способ распознавания речи

## Процесс решения

В функционале сервиса предполагается внедрение технологий по распознаванию текста из голоса. Создание такого функционала может существенно упростить процесс ввода информации в некоторые документы, заполняемые экспертами на мерориятиях, тем самым экономя суммарное время на заполнение одного документа.

Для С++ существует несколько способов по распознованию речи в текст:
1. Whisper
2. Echo
3. Cloud Speech-to-Text API
4. Julius
5. RWTH ASR
6. HTK 
7. Kaldi 
8. Yandex Speech kit

Среди перечисленных инструментов по распознованию голоса выделяют несколько кандидатов: **Kaldi** и **Whisper**. Каждый из вариантов имеет недостатки. 
**Julius**. Разработан на языке С в Японии и на данный момент широкое распространение получил только вариант для распознавания японского языка. Доступно также распознавание английской речи, однако поддержка русской речи отсутствует, поэтому данный вариант не подойдёт для проекта.
**CMU Sphinx**. Состоит из различных модулей для извлечения речевых признаков, распознавания речи (в том числе и на мобильных устройствах) и обучения такому распознаванию. Из минусов использования данной системы можно выделить невозможность запустить CMU Sphinx «из коробки», т.к. даже для решения простых задач требуются знания по адаптации акустической модели, в сфере языкового моделирования.     
**RWTH ASR**. В проект также входят документация по установке, различные обучающие системы, шаблоны, акустические модели, языковые модели, поддержка нейронных сетей и т. д. При этом RWTH ASR практически не обладает кроссплатформенностью и имеет низкую скорость работы.
**HTK**.  Несмотря на то, что исходный код является общедоступным, использование HTK для создания систем для конечных пользователей запрещено лицензией, из-за чего инструментарий сейчас не является популярным. Также система имеет относительно невысокие скорость и точность работы.
**Whisper**. Инструмент, созданный в 2022 году OpenAI. Обучен на 680.000 часов аудиозаписей, устойчив к акцентам и фоновым шумам. Для стабильной работы требует достаточно высокие мощности у оборудования, что делает вариант его использования не самым удобным.
**Kaldi**. Проект с открытым исходным кодом, написанный на C++. Выделяется относительной быстротой работы и низкими требованиями к оборудованию для стабильной работы.
**Yandex SpeechKit**. Инструмент в первую очередь для Python. Есть возможность использования API.

В первую очередь опробуем **Kaldi** для распознавания русской речи в текст для проекта.
