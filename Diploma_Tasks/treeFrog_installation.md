# Задача №N - установка 
## Процесс решения

TreeFrog — это высокопроизводительный полнофункциональный (full-stack) веб-фреймворк, построенный на базе библиотеки Qt. Реализует классическую модель «Модель-Вид-Контроллер», обеспечивая четкое разделение логики и представления. Приложения работают быстрее аналогичных решений на интерпретируемых языках благодаря компиляции в нативный код и оптимизированному движку сервера.
Минимизирует объем конфигурационных файлов, используя стандартные соглашения об именовании. Активно использует неграфические модули Qt (контейнеры, сеть, SQL, JSON, метаобъекты). Упрощает работу с базами данных (MySQL, PostgreSQL, SQLite, Oracle, MongoDB и др.), скрывая сложные SQL-запросы.

Система шаблонов: Использует ERB-подобный движок (как в Ruby on Rails) для генерации HTML-страниц.
WebSocket: Полноценная поддержка протокола для двусторонней связи в реальном времени.
Генератор (Scaffolding): Автоматическое создание каркаса приложения (моделей, контроллеров и Makefile) одной командой.
Кроссплатформенность: Работает на Windows, macOS и Linux. 
Фреймворк идеален для разработчиков, знакомых с экосистемой Qt, которым требуется создавать высоконагруженные веб-сервисы, API или встроенные веб-интерфейсы для устройств с ограниченными ресурсами. 

Несерьёзными техническими особенностями фреймворка являются поддерживаемые версии Qt: 6.8 и 6.9. А также не поддерживается mingv. Фреймворк создан для компилятора Visuaal Studio. Установку фреймворка можно исправить, путём изменения ***qtenv2.bat***, внутри которого записаны пути для обращения в системную переменную path. 
```
@echo off
echo Setting up environment for Qt usage...
set PATH=C:\Libs\Qt\6.10.0\mingw_64\bin;C:\Libs\Qt\Tools\mingw1310_64\bin;%PATH%
cd /D C:\Libs\Qt\6.10.0\mingw_64
```
Изменив пути, можно сделать возможным установку фреймворка для более старшей версии. После установки нужно будет удостовериться в работоспособности фрейморка на версии 6.10.
При попытках работы в таких условиях фреймворк отказался устанавливаться на персональный компьютер. Данная особенность связана с тем, что фреймворк проверяет версии файлов перед установкой и при нахождении несоответствий в версиях блокирует установку.
После установки версии 6.8.3 Qt установка прошла успешно. Был собран стандартный проект с возможностью запуска на локальном хосте. В качестве инструмента для управления базой данных выбран sqlite3 и программа dbeaver с удобным пользовательским интерфейсовм, позволяющим без лишних манипуляций в командной строке управлять базой данных.

Создав стандартный проект по приложенной инструкции в документации фреймворка, открывается возможность его модификации под рабочие задачи, решаемые проектом. Для повышения удобства работы с файлами контроллеарми, ERB-шаблонов и заголовочными, необходима доработка проекта внутри среды разработки. По скольку фреймворк направлен на работу с С++ была выбрана Visual Studio. Для сброки файлов решений предварительно необходимо подключить зависимости между файлами проекта и файлами фреймврока, что не делается по умолчанию. После подключения необходимых зависимостей становится возможным беспрепятственная работа.

Для работы над внешним видом пользовательского интерфейса были выбраны поддерживаемые TreeFrog шаблоны в формате ERB, позволяющие создавать программный код на С++ внутри шаблона, после чего тот свободно компилируется в dll для отрисовки HTML страниц. Существенным минусом в работе с данным видом шаблонов является необходимость обязаельность перекомпиляции после внесения в шаблон каких-либо изменений для пересборки библиотек для отрисовки страниц. Имеется поддержка CSS и JS файлов для разработки сложных информационных систем. Благодаря такому подходу упрощается работа с программныи кодом форм и написание бизнес-логики.

```
1  <body>
2     <% tfetch(QString, newTitle); %>
3     <h1><%= newTitle %></h1>
4     <div class="reportContainer grid-form">
5        <%== formTag(urla("submitForm"), Tf::Post) %>
6  
7        <div class="field">
8          <label>Полное название клуба</label>
9          <input type="text"    name="NamePutOfTheClubHere">
10       </div>
11 
12       </form>
13    </div>
14 </body>
```

Специальные тэги, как в строке №2 позволяют передавать через контроллеры данные для их отображения на конечной странице. Тэг на строке №5 определяет тип запроса HTTP и метод из контроллера для исполнения при исполнении запроса типа Post. Данная конструкция является универсальной для разрабатываемого проекта и будет использоваться в основе каждой из пользовательских форм, приближённо повторяющих шаблоны документов от РКФ. Сами файлы контроллеров представляют собой связанные файлы cpp и h расширения с классами.

При переносе прототипа на фреймворк опытным путём выяснилось, что использую тэги форм можно отказаться от модели данных, созданной ранее для сохранения вводимой информации. Пользуясь стандартными возможностями форм для браузеров, нет необходимости разделять документы на несколько страниц. В добавок, внутри браузеров есть возможность возвращения обратно на только что покинутую страницу, при этом все введённые данные сохраняются внутри полей форм, что обеспечивает возможность внесения изменений при обнаружении ошибок.

Проект подразумевает возможность заполнения нескольких видов шаблонов, утверждённых ранее РКФ. Данные условия делают необходимым уход от жёсткого кодирования путей к файлам скриптов и шаблонов word. Достигнуть цели по унификации работы со скриптами и шаблонами необходимо разработать файл с настройками, которым будет следовать проект при работе с разными шаблонами, передавать необходимую информацию для выбора нужного шаблона и последующего заполнения.

В результате переноса прототипа на TreeFrog изменениям была подвержена вся бизнес-логика с точки зрения программного кода. Ранее созданный в QtCreator прототип был создан силами среды: пользовательские формы. Для работы с новые формами, располагающимися в браузере коды проекта претерели изменения, но конечный результат не ихменился. Логика работы была перенесена в контроллеры, которые не знают о существовании пользовательской формы ERB шаблона. Регулярные выражения для контроля ввода пользовательской информации имеют возможность быть помещёнными внутрь тэгов ERB шаблона.
