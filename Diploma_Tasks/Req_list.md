# Задача №1 - составление функциональных и нефункциональных требований

## Процесс решения

### Причины составления:
Первопричина для составления чёткого списка требований к системе - систематизация предстоящего процесса разработки. Без явного списка сформулированных требований нет ясных ориентиров для разработки. Список требований содержит критерии и рамки для разработчика, помогая достичь поставленной цели и не выходить за пределы поставленных задач в реализации функционала. В ходе общения с экспертами из РКФ был составлен следующий список требований, который отражает миннимальный набор условий, которым должна соответствовать готовая информационная система:

### Неформазилованные требования:
1. Система должна предоставлять возможность заполнения бланков кинологических мероприятий для последующей печати и простановки подписи;
2. Система должна сохранять прогресс заполнения документа.
3. Система должна выдавать файл на скачивание не позже 1 минуты после завершения заполнения;
4. Система должна предоставить возможность прекращения заполнения документа при необходимости без нужды в сохранении файла с уже заполненными полями;
5. Система должна предоставлять функционал по заполнению бланков только авторизованному пользователю;
6. Система должна проверять пароль на соответсвие парольному стандарту и отклонять слишком простые варианты;
7. Система не должна хранить персональные данные пользователей в открытом виде, либо вовсе не хранить таковых;
8. Система должна сохранять информацию в полях после первой выдачи готового файла для возможности редактирования информации и составления нового файла с исправленными ошибками;
9. Система должна предоставить возможность очистки текстовых полей по нажатию кнопки;
10. Система должна предоставить возможность обработки голоса пользователя и перевода речи в текст при заполнении полей;

Составленный список неформализованных требований необходимо для удоства дальнейшей работы разделить на функциональные требования и нефункциональные. Данное разделение поможет точнее понимать требования и применять их конкретно к внутренней и внешним частям системы.

### Функциональные требования 
- Ф-1: Система должна предоставлять возможность заполнения бланков кинологических мероприятий для последующей печати и простановки подписи;
- Ф-2: Система должна сохранять прогресс заполнения документа при выходе из браузера и предлагать продолжить заполнение после повторного захода при условии сохранения открытой вкладки;
- Ф-3: Система должна предоставить возможность прекращения заполнения документа при необходимости без нужды в сохранении файла с уже заполненными полями;
- Ф-4: Система должна предоставлять функционал по заполнению бланков только авторизованному пользователю;
- Ф-5: Система должна сохранять информацию в полях после первой выдачи готового файла для возможности редактирования информации и составления нового файла с исправленными ошибками;
- Ф-6: Система должна предоставить возможность очистки текстовых полей по нажатию кнопки;

### Нефункциональные требования
- НФ-1: Система должна выдавать файл на скачивание не позже 1 минуты после завершения заполнения;
- НФ-2: Система не должна хранить персональные данные пользователей в открытом виде, либо вовсе не хранить таковых;
- НФ-3: Система должна проверять пароль на соответсвие парольному стандарту и отклонять слишком простые варианты;
- НФ-4: Система должна предоставить возможность обработки голоса пользователя и перевода речи в текст при заполнении полей;
