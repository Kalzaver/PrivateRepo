# Задача №N - продумывание возможно развития проекта 
## Процесс решения

Разрабатываемый сервис имеет несколько возможных путей развития и не исключает их совместное создание в теле одной разработки. Функционал по внесению определённой информации в файлы Word имеет безграничный спект использования: от составления экзаменационных билетов для очной сдачи, до заполнения других документов. Кроме этого, при заполнении документов создаётся "цифровой двойник" (далее ЦД), чья область приминения также обширна. При сборе информации с таких ЦД можно реализовать анализ данных: классификация, кластеризация - для построения диаграмм. Например для учёта популярности пород, которые проходят то или иное мероприятие. 
ЦД судьи в теории может собирать информацию о том, какие документы он создавал. 

В стандартные возможности фреймворка входит поддержка разнличных типов баз данных и хранилищ информации. В него входят такие известные БД как MongoDb, Sqlite, Postgre и другие. Фреймворк в состоянии работать с реляционными и нереляционными БД, что позволяет разрабатывать разнообразные сервисы и платформы.

Для проекта была выбрана БД Sqlite. Во-первых, при необходимости смены БД на другую, импортировать информацию в другую из нынешней не составит особых трудностей. Во-вторых, данная БД не требует для своей работы отдельного сервера, ни виртуального, ни физического. В-третьих, данная БД проста в освоении.

В качестве начальной версии базы данных бюла создана БД для хранения части информации из документа по племенному смотру, так как в нём есть информация, при помощи которой можно классифицировать ту или иную породу собаки, а так же применить другие методы обработки данных при заполнении людьми документа.

```
1  CREATE TABLE breed_data (
2      id INTEGER PRIMARY KEY AUTOINCREMENT,
3      name_of_the_breed TEXT,
4      put_weight_here REAL,
5      put_hieght_here REAL,
6      put_here REAL,
7      put_elbow_here REAL,
8      put_depth_here REAL,
9      put_chest_here REAL,
10     put_circimference_here REAL,
11     put_head_length_here REAL,
12     put_muzzle_here REAL,
13     put_skull_here REAL,
14     put_founding_here REAL,
15     created_at DATETIME
16 );
```

Приведённый выше SQL-скрипт создаёт базу без персональных данных владельцев, что упрощает обработку инфорамцииЮ исключая необходимость шифрования или защиты таких данных, что не нарушает законодательства Российской Федерации. Каждая новая запись содержит свой собственнный id (2 строка), а также дату создания (15 строка). Данные имеют тип REAL, который используется для хранения чисел с плавающей точкой, так как измеряемые параметры каждой собаки могут быть не ровными и тип INTEGER им не подходит.

Сама БД создаваль при помощи отдельного ПО -  Dbeaver - удобной СУБД, способной работать с многими видами БД.
![Как выглядт интерфейс управления базой данных](//Diploma_Tasks/Images/Dbeaver.png)

С каждого отправляемого документа часть информации добавляется в БД. Сбор информации реализован в том же контроллере, который отвечает за сбор информации для передачи её скрипту замены на Python. 

```
1  QString breed = data.value("NameOfTheBreed").toString();
2      double weight = data.value("PutWeightHere").toString().toDouble();
    ...
3      double founding = data.value("PutfoundingHere").toString().toDouble();
4
5      BreedData::create(breed, weight, height, puthere, elbow, depth,
6          chestaround, cirsimaround, headlength, muzzle, skull, founding);
```
