# Задача - выбор типа архитектуры для веб-сервиса

## Процесс решения

Выбор архитектуры - отправная точка для создания кодовой составляющей системы. Главным образом выбор лежит между двух вариантов: микросервисы и монолит.

Основное преимущество микросервисной архитектуры является в способности системы к естественному масштабированию. Микросервисы работают отдельно друг от друга, передавая друг другу необходимую информацию. В то же время монолит очень зависимая вещь, которая не поддерживает сильные изменения в себе. Зачастую изменения в монолите отражаются на всей системе и требуют большего контроля для корректного внедрения. Для проектируемой системы будет предпочтительна.

Гибкость микросервисной архитектуры открывает уникальные возможности для технологического разнообразия. Ядро системы, отвечающее за обработку PDF-документов, может быть реализовано на C++ с использованием специализированных библиотек, что обеспечивает максимальную производительность при работе с сложными документами. В то же время веб-интерфейс будет использовать более подходящие для этих задач технологии, создавая тем самым оптимальную технологическую экосистему, где каждый инструмент применяется согласно своей задаче.

Устойчивость к отказам неотъемлемое качество, делающее микросервисную архитектуру предпочтительной для веб-сервиса. В традиционных монолитных системах незначительная ошибка в одном модуле может привести к коллапсу всей системы. В микросервисной архитектуре неполадки, например в сервисе уведомлений или временная недоступность сервиса аутентификации, не остановят процесс заполнения документов и система продолжит работу, изолировав проблемный компонент до момента его восстановления.

Вносить изменения в систему, построенную на микросервисах, проще чем в монолит, так как монолит не предполагает добавления нового функционала за короткое время. Для внедрения необходимо вносить изменения во всю систему и грамонто вносить новый функионал в уже существующий.

В конечном итоге для разрабытываемого веб-сервиса более подходящей видится архитектура типа микросервисов.
